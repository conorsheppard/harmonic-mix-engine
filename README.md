## Harmonic Mix Engine

A Spring Boot backend and Next.js frontend, packaged as Docker images and runnable locally on Kubernetes via Minikube.

### Prerequisites
- Minikube
- kubectl
- Docker
- Make

### Building

Build both Docker images:
```shell
make build-all
```

Or individually:
```shell
make build            # backend
make build-frontend   # frontend
```

### Running the Application

```shell
make
```

This deploys to Minikube and port-forwards both services:
- Backend: http://localhost:8080
- Frontend: http://localhost:3000

Hit the harmonic mixing endpoint:
```shell
curl http://localhost:8080/songs?key=C
```

### Cleanup

```shell
make cleanup
```

## Deploying to Azure (Terraform)

The `terraform/` directory provisions a single Ubuntu VM (`Standard_B2ls_v2`,
West Europe) that runs the app on Minikube behind an Nginx reverse proxy on
port 80 — cloud-init installs Docker/Minikube/kubectl/Nginx, clones this repo,
and runs `make azure-vm-up` on first boot, so one `terraform apply` yields a
live app. State is kept in an Azure Storage blob so you can manage it from
multiple machines.

### Prerequisites
- Terraform >= 1.5
- Azure CLI, logged in: `az login`

### One-time: remote state backend

Creates the storage account/container for Terraform state and writes
`terraform/config.azurerm.tfbackend` (commit it — the account name isn't a secret):

```shell
bash scripts/terraform/bootstrap-remote-state.sh
```

### Provision

```shell
cd terraform
cp terraform.tfvars.example terraform.tfvars   # set subscription_id
terraform init -backend-config=config.azurerm.tfbackend
terraform apply
```

Cloud-init takes a few minutes after apply completes. Then:

```shell
curl "$(terraform output -raw app_url)"        # http://<ip>/songs?key=C
```

### SSH access (on any machine)

The keypair is generated by Terraform and stored in state. Retrieve the private
key locally:

```shell
terraform output -raw ssh_private_key > ../.priv/harmonic-mix-engine-vm_key.pem
chmod 400 ../.priv/harmonic-mix-engine-vm_key.pem
terraform output -raw ssh_command
```

### Second machine

```shell
az login
cd terraform
terraform init -backend-config=config.azurerm.tfbackend   # shares remote state
terraform plan                                            # should show no changes
```

### Tear down

```shell
terraform destroy
```
